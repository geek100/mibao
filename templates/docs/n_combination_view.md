N组合视图(适用于原生活动区域展示)
-----
###### 1.概述
```text
1.app开发时经常会遇到某一块视图变动很频繁;
2.如首页某一区域需要做活动;
3.在满足以上1和2的同时需要改变视图的呈现方式;

计划:
   第一阶段每个占位用图片来呈现;
   第二阶段可在每个占位上增加一些修饰视图;
```
###### 2.呈现形式
```java
 --------------------------------           ------------------------------------    
|              |      |          |         |               |                    |   
|              |      |          |         |               |                    |   
|--------------------------------|         |               |--------------------|   
|                  |             |         |---------------|                    |   
|                  |             |         |               |                    |   
|                  |             |         |               |--------------------|   
|                  |             |         |               |                    |        ......
 --------------------------------           ------------------------------------    
```
###### 3.1第一阶段——每个占位数据格式及属性描述
```json
{
  "width": 200,
  "height": 80,
  "orientation": "h/v",
  "block": {
    "imgUrl": "",
    "bg": "#ffffff"
  }
}

width:每个单元宽度
height:每个单元高度
orientation:h-横向排版(即，不同单元横向组合);v-纵向排版(即,不同单元纵向组合);
//块数据
block:{
	imgUrl:块图片url,做为背影填充整个块,
    bg:块背影(第二阶段用)
}
```
###### 3.2第二阶段——每个占位增加的数据
```json
	待定
```
###### 4.实现及交互逻辑
```java
1.布局方式：根据orientation递归数据安层级构建视图;
2.每个块第一阶段用图片来填充;
3.每次递归最后一个视图根据总宽(高)度-前面已构建视图宽(高)度=最后一个视图宽(高)度
......
```
